收集一些比较有用但是用的不太多的git命令

  1. 暂时将未提交的变化移除，稍后再移入

  1. 将master分支的文件以zip的压缩格式打包成/path/to/file.zip文件

  2. 产生一个新的commit，用来抵消指定的commit所做的修改

COMMIT 为想要抵消的提交版本号，通常用在错误提交的回滚

  1. 检出远程的一个指定分支

  2. 新建一个没有历史记录信息的分支

  3. 忽略一个可能被频繁修改的文件，在merge发生时这个文件将会保持不改变

  4. pull时消除无用的日志信息
